name: explain
description: Explain code — what it does, its architecture, patterns, and edge cases
allowed-tools: Read, Glob, Grep
---

Explain the code at $ARGUMENTS concisely. Cover:

- **What it does**: Summarize the purpose and logic flow in a few sentences
- **Architecture**: Note design patterns, key dependencies, and how it fits into the broader codebase
- **Edge cases**: Highlight any notable edge cases, error handling, or potential pitfalls

Keep the explanation short and scannable. Use bullet points. Avoid restating the code line-by-line — focus on intent and the "why" behind decisions.
